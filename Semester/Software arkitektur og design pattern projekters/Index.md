# **Hvad er softwarearkitektur?**

Softwarearkitektur beskriver den overordnede struktur i et system — hvordan kode, domæne, data og brugergrænseflade organiseres, og hvordan de forskellige dele samarbejder.

Arkitektur handler ikke om individuelle klasser, men om **systemets form**, **ansvarsfordeling**, og **hvordan afhængigheder styres**. En god arkitektur gør det muligt at ændre, teste og udvide systemet uden at skabe unødvendig kompleksitet.

I dette projekt bruges arkitektur til at skabe et klart skel mellem forretningslogik, applikationslogik, tekniske implementeringer og UI, så systemet bliver mere robust, fleksibelt og let at vedligeholde.
